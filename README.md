## Descripción

Aplicación que estoy desarrollando para facilitar el manejo de archivos de guardado de videojuegos, permitiendo la **coexistencia de múltiples partidas** o "Perfiles" dentro de un mismo juego y guardando un **historial de checkpoints** para poder volver a un estado previo.

Para usarlo es necesario primero configurar una "Whitelist" y "Blacklist" con los directorios que el programa puede o no modificar. Una vez hecho esto, se registra un juego en el programa añadiendo su titulo y los lugares donde se almacenan su archivos de guardado, se crea una partida y guarda un checkpoint dentro de esa partida para registrar el estado de esa carpeta de guardado en ese punto. 

Una vez guardado el estado de la carpeta en un checkpoint, es posible revertir en cualquier momento el estado de la carpeta al momento en el que se guardo el checkpoint, o exportar los archivos de la carpeta en un ZIP para cargarlos manualmente si se quisiera. La existencia de multiples partidas permite agrupar checkpoints relacionados entre sí. Si hay 2 partidas, A y B, y la ultima partida cargada fue A, en caso de que se quiera cargar un checkpoint de B, se creará un punto de guardado automatico en A.

Se permite definir estrategias para al rótacion de checkpoints, es decir, si se iran eliminando los mas viejos a medida que pasa el tiempo para ahorrar espacio y de que manera. Actualmente hay 3 opciones: Sin eliminacion de checkpoints antiguos, con eliminacion modo FIFO estableciendo un maximo de checkpoints, y eliminacion aleatoria, donde se eliminan los checkpoints al azar, teniendo los mas viejos mas posibilidad de ser elimiandos, y permitiendo garantizar un rango en el que nunca se eliminan y un maximo de checkpoints en simultaneo tras el cual se eliminará siempre alguno. Tener en cuenta que puede que haya que guardar un checkpoint para que este algoritmo surta efecto.

Se recomiendo cerrar el juego al momento de guardar o cargar archivos, así como no tener abierto ninguno de sus archivos con algun programa. Tener en cuenta que al momento de cargar un checkpoint, cualquier archivo que no haya estado en el directorio cuando se cargo el checkpoint será eliminado, por lo que es importante no cambiar el directorio por error a uno que tenga informacion importante que no se deba borrar, ni guardar otros archivos en una carpeta de guardado registrada.

## Version 0.1.0 - Modelo Cliente-Servidor
Originalmente, el programa iba a tener un módulo "**Servidor**" que se iba a encargar de manejar la base de datos con las partidas guardadas, y un módulo "**Cliente**" que va a permitir conectarse al módulo servidor local para cargar y descargar archivos de guardado según indicado por el usuario, o conectarse con módulos Servidor externos para obtener archivos de guardado de otros nodos, facilitando la transferencia de archivos entre computadoras para facilitar la continuidad al jugar una misma partida a través de distintos dispositivos. La idea era que se pueda mantener una instancia de "servidor" iniciada permanentemente en una computadora para poder "pedir" los archivos.

Tras repensarlo, decidi que sea un módulo unico que deba iniciarse en ambos nodos al momento de realizar la transferencia. Se puede hallar una version semi-funcional del modelo anterior en la version 0.1.0, commit **302e74e**
